\section{When to Use Agentic AI}

% ---- Slide: Strategy Creation, Strategy Execution ----
\begin{frame}{When to use Agents: Strategy Creation - Chatbot Like Experience}

Given a problem, agents are used to \textbf{create or refine a plan/strategy} when the solution path is uncertain and it requires skills/knowledge tools we do not have, or the knowledge is spread across different data sources and data types. The agent can navigate the information available, spread across different data sources and types, and leverage the FM to connect the docs and answer the user query.

\newline
Example of application:

\begin{enumerate}
  \item Enterprise Knowledge Assistant
  \item Regulatory Compliance BOTS
  \item Customer Support copilots
  \item Developer Documentation Agents
  \item Data analyst Agents
\end{enumerate}
\end{frame}

% ---- Slide: Decision Making - Executing the Strategy - Deterministic Code vs Runtime Agent ----
\begin{frame}{When to Use Agentic AI - Decision Making: Strategy Execution}
After planning, execution can follow different paths:
\vspace{0.4em}
\begin{columns}[T]
\begin{column}{0.48\textwidth}
\begin{block}{Case 1 — Deterministic Execution}
\begin{itemize}
  \item Plan is well specified
  \item Environment is stable
  \item Limited unforeseen events
  \item Output: scripts, workflows, decision trees
  \item Use the coding agent to automate the workflow
\end{itemize}
\end{block}
\end{column}
\begin{column}{0.48\textwidth}
\begin{block}{Case 2 — Runtime Agentic Execution}
\begin{itemize}
  \item Plan remains high-level
  \item Environment is dynamic / non-stationary
  \item System must adapt step by step
  \item Output: ongoing reasoning + actions
  \item Use a long-running autonomous agent (can use the coding agent as a tool)
\end{itemize}
\end{block}
\end{column}
\end{columns}
\vspace{0.5em}

\end{frame}
