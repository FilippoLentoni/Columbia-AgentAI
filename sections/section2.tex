% ---- Slide: Action Layer ----
\begin{frame}{Action Layer}

\textbf{Action Layer}
\begin{itemize}\setlength\itemsep{0.2em}
  \item \textbf{Tools}: callable capabilities that let the agent interact with the world. MCP is the open standard that lets AI applications connect in a consistent way to external tools.
  \item \textbf{Skills}: reusable workflows that combine reasoning and often multiple tools
\end{itemize}

\vspace{0.4em}

\textbf{Tool examples}
\begin{itemize}\setlength\itemsep{0.15em}
  \item \texttt{search\_web(query)}: retrieves recent information from the web
  \item \texttt{execute\_sql(query)}: queries a database and returns structured results
\end{itemize}

\textbf{Skill examples}
\begin{itemize}\setlength\itemsep{0.15em}
  \item \textbf{Customer support triage}: read the ticket, classify intent, retrieve the order status, draft a response, and escalate if needed
  \item \textbf{Weekly business review generator}: pull metrics from dashboards, summarize deltas, identify anomalies, and draft the report
\end{itemize}

\vspace{0.2em}
A tool is a single callable capability; a skill is a reusable workflow that combines reasoning and often multiple tools.

\end{frame}

% ---- Slide: RAG — Two-Phase Pipeline ----
\begin{frame}{Tool Example - RAG: Retrieval-Augmented Generation}
\textbf{Goal}: ground model outputs in external knowledge beyond the training cutoff\\[0.4em]

\vspace{0.4em}
\begin{columns}[T]
\begin{column}{0.48\textwidth}
\begin{block}{Phase 1 — Offline: Knowledge Base Preparation}
\begin{enumerate}
  \item Ingest documents
  \item Chunk text \textit{(strategy, size, overlap)}
  \item Compute embeddings \textit{(model choice)}
  \item Store in vector index \textit{(distance metric)}
\end{enumerate}
\vspace{0.2em}
\end{block}
\end{column}
\begin{column}{0.48\textwidth}
\begin{block}{Phase 2 — Online: Retrieval \& Generation}
\begin{enumerate}
  \item Embed user query
  \item Retrieve top-$k$ chunks 
  \item Inject context into prompt
  \item Generate grounded answer 
\end{enumerate}
\end{block}
\end{column}
\end{columns}
\vspace{0.4em}

\end{frame}

% ---- Slide: Memory Layer ----
\begin{frame}{Memory Layer}

\begin{columns}[T]
\begin{column}{0.48\textwidth}
\textbf{Short-Term Memory}
Short-term memory captures turn-by-turn interactions within a single session. This lets agents maintain immediate context without requiring users to repeat information.

\vspace{0.3em}
\textit{Example:} If a user asks, ``What's the weather like in Seattle?'' and then follows up with ``What about tomorrow?'', the agent uses the recent conversation history to understand that ``tomorrow'' still refers to Seattle.
\end{column}
\begin{column}{0.48\textwidth}
\textbf{Long-Term Memory}\\[0.3em]
Long-term memory automatically extracts and stores key insights across multiple sessions, including user preferences, important facts, and session summaries.

\vspace{0.3em}
\textit{Example:} If a customer mentions they prefer window seats during flight booking, the agent stores that preference. In future interactions, it can proactively offer window seats and create a more personalized experience.
\end{column}
\end{columns}

\vspace{0.5em}
\begin{block}{Why Memory Matters}
Without memory, every conversation starts from zero - customers repeat themselves, context is lost, and personalization is impossible. Memory enables agents to \textbf{recognize returning users}, \textbf{recall preferences automatically}, and \textbf{connect related conversations} over time.
\end{block}
\end{frame}


% ---- Slide: Safety Layer ----
\begin{frame}{Safety Layer}

\small
Guardrails operate on agent inputs and outputs and provide configurable safeguards to detect and mitigate risks such as:

\begin{itemize}\setlength\itemsep{0.2em}
  \item \textbf{Prompt Injection}: malicious instructions that try to override system prompts or intended behavior
  \item \textbf{Jailbreaks}: adversarial prompts that attempt to bypass safety safeguards
  \item \textbf{PII Exposure}: sensitive personal data appearing in inputs, memory, or outputs
  \item \textbf{Toxicity}: harmful, abusive, or hateful language in prompts or responses
  \item \textbf{Prompt Leakage}: disclosure of hidden prompts, internal instructions, or private context
\end{itemize}

\vspace{0.4em}

To further improve the safety of AI applications, developers can also control tool authorization and permissions through custom policies at the user level, ensuring that agents can access only the tools and actions allowed for a given user or role.

\end{frame}
